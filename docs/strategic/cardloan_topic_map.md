## 1. ドメイン定義

soico の cardloan/ カテゴリは、URL 上は "cardloan" だが、実態は「個人を中心に法人を含めた幅広い資金調達のハブ」として運用されている。本ドキュメントはこの実態に合わせて構造を定義する。

## 2. 二次元構造

cardloan 配下の記事は **(subtopic 軸) × (V軸: 商品/サービス軸)** の二次元セルに配置される。

### 2.1 概念階層 vs 実装階層 (重要)

- **概念階層** (なぜ存在するか):
  - C: 目的 (ユーザー視点) > B: 条件 (供給側視点)
  - A: 属性 は制約条件
- **実装階層** (実際の記事/KWの実態):
  - A, B, C, E, F, G が並列ノード
  - 検索者は B (即日) で検索することも C (医療費) で検索することもある
  - Google も別 KW として扱う
  - SEO 実装上は並列の subtopic として扱わざるを得ない

`master_topics` は実装階層で持ち、概念階層はメタ情報として保存する。

## 7. カニバリ判定への含意

二次元セル構造により、以下のパターンで判定する:

```
セル内重複 (同 subtopic × 同 V軸)
  → 強カニバリ確定 (統合候補)
  例: (A3 主婦) × (V1 カードローン) に5記事 → 統合候補

セル横断重複 (同 subtopic × 異 V軸)
  → SERP 重複度で判定
  例: (B1 即日) × (V1 カードローン) と (B1 即日) × (V2 キャッシング)
  → SERP 重複高 → 統合候補
  → SERP 重複低 → 別記事として両立

異セル (異 subtopic × 同 V軸)
  → 正常関係 (同一商標で異なるテーマの記事)
  例: (V5-3 アコム) × (B2 在籍確認) と (V5-3 アコム) × (F5 返済)

異商品の正常関係 (異 subtopic × 異 V軸)
  → カニバリでない (別個別記事)
```

---

## 8. 重要原則

### 8.1 SERP 重複 ≠ カニバリ
意図一致が前提。意図が異なれば「差別化」が解、「統合」ではない。Decision engine の設計に反映する。

### 8.2 二次元構造の利点
URL を変えずに metadata (subtopic_axis / vocabulary_axis) を変更するだけで論理的な再分類が可能。物理移動なし。

### 8.3 検索意図3層は別レイヤー
- subtopic × V軸 = 記事間関係 (本ドキュメントの範囲)
- 検索意図3層 (顕在/潜在/安心) = 記事内構造の評価軸 (自走リライトシステムの範囲)
- 混同せず分離して扱う。
