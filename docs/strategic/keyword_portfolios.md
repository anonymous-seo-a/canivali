## 1. KW 生成ルール (4層テンプレート)

### Layer 1: Atomic (最小構成 KW)

[V軸名] そのもの / [subtopic name] そのもの

例:
- "カードローン" / "即日融資" / "在籍確認" / "アコム"

### Layer 2: Composition (subtopic × V軸 の合成)

[V軸] × [subtopic]

例:
- "アコム 在籍確認"
- "主婦 カードローン"
- "即日 アコム"

### Layer 3: Modifier (修飾語の付与)

修飾語カテゴリ:

| カテゴリ | 修飾語例 |
|---------|---------|
| 疑問形 | とは / 方法 / 仕組み / 流れ / 違い |
| 評価形 | おすすめ / 比較 / ランキング / 口コミ / 評判 |
| 可否形 | 借りれる / 通る / 落ちる / なし / OK |
| 否定/不安形 | 違法 / 危険 / 安全 / 厳しい / 甘い |
| 数値形 | ○万円 / ○歳 / ○社目 |
| 状況形 | 知恵袋 / 今すぐ / 急ぎ / 土日 / 夜間 |
| バレ対策形 | バレない / 知られない / 郵送物なし / 電話なし |

### Layer 4: Multi-axis (複数軸の交差)

[A: 属性] × [B: 条件] × [V: 商品]

例:
- "主婦 即日 カードローン"
- "学生 在籍確認なし アイフル"

**制約**: 全組み合わせは生成しない。SerpAPI で実検索ボリュームのあるもののみ採用。

## 2. 修飾語の subtopic 帰属マッピング

| 修飾語 | 帰属 subtopic |
|--------|--------------|
| "おすすめ" / "比較" / "ランキング" | D (ピラー) |
| "とは" / "仕組み" / "違い" | E (教養) |
| "方法" / "流れ" / "手順" / "借り方" | F (実務) |
| "借りれる" / "通る" / "甘い" / "厳しい" | A (属性) or B (条件) — 文脈分岐 |
| "バレない" / "知られない" / "郵送なし" | B (バレ対策スペック) |
| "払えない" / "滞納" / "踏み倒し" | G (トラブル) |
| "違法" / "危険" / "詐欺" | G (違法被害) or 警告 |

## 3. KW → セル逆引き分類

記事タイトル/H1 から、修飾語 + V軸トークンで自動的に (subtopic, V軸) を推定する。

```python
def kw_to_cell(kw: str) -> tuple[str, str]:
    v_axis = match_v_token(kw)              # 商標・商品名を検出
    subtopic = match_subtopic_modifier(kw)  # 修飾語からsubtopic推定
    return (subtopic, v_axis)
```

Phase 1 で `master_articles` のセル割当エンジンとして実装。

---

## 4. 同義語正規化

`master_keywords.canonical_keyword_id` で表記揺れを統合する。

例:
- "楽天銀行スーパーローン" / "楽天スーパーローン" / "楽天 カードローン" → 同一 canonical (V6-4)
- "アイフル" / "AIFUL" → 同一 canonical (V5-1)
- "在籍確認" / "在席確認" / "勤務先確認" / "在確" → 同一 canonical (B2)

---

## 5. 検索意図3層との関係 (重要)

KW portfolio は **subtopic × V軸 の二次元** である (記事間関係)。
検索意図3層 (顕在/潜在/安心) は **記事内構造の評価軸** である (記事内深度)。

両者は別レイヤーで、混同せず分離して扱う。
- KW portfolio 生成では3層は意識しない
- 記事生成 (リライト時) では3層を意識する

## 7. 他セルへの展開ルール

Phase 1 で `master_keywords` テーブルに以下を一括登録する:

1. Layer 1, 2 は subtopic × V軸 の組み合わせから機械生成
2. Layer 3 は修飾語マッピングから自動付与
3. Layer 4 は SerpAPI で検索ボリューム確認後にフィルタ
4. canonical_keyword_id は同義語クラスタリングで自動付与

総量見積:
- 平均 22 主軸 KW × 200 セル ≈ 4,400 KW
- 移譲対象を含めると約 6,000 KW
- GSC 実績 KW (約 11,870 件) と突合してマッチング
